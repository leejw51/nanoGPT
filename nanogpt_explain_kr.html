nanoGPT, 처음부터 이해하기

GPT 하나를 학습하고 실행하는 약 600줄짜리 저장소입니다. 핵심은 두 파일이 담당합니다. model.py는 신경망을 정의하고, train.py는 그것을 최적화합니다. 이 페이지는 설치 → 데이터 → 학습 → 샘플링까지의 전 과정을 따라가며, 모든 텐서의 모양(shape)을 설명하고, 각 단계의 모양을 출력하려면 정확히 무엇을 수정해야 하는지 보여줍니다.

PyTorchGPT-2 아키텍처~10.6M–1.5B 파라미터 문자 또는 BPE 토큰단일 파일 모델

1 · 개요

nanoGPT는 한 번에 다 읽을 수 있는 디코더 전용(decoder-only) 트랜스포머(GPT-2 아키텍처)입니다. 모델의 역할은 아주 단순합니다. 토큰 시퀀스가 주어지면 다음 토큰을 예측한다. 이것만 잘하면, 토큰을 한 번에 하나씩 샘플링해 다시 입력으로 넣어 텍스트를 생성할 수 있습니다.

1. 데이터 준비

data/*/prepare.py

원시 텍스트를 정수 토큰 id의 흐름으로 바꿔 train.bin / val.bin으로 저장합니다.

2. 학습

train.py + model.py

반복: 배치를 가져오고, 다음 토큰을 예측하고, 오차(loss)를 측정하고, 가중치를 조금씩 조정합니다.

3. 샘플링

sample.py

학습된 가중치를 불러와 자기회귀적으로 새 텍스트를 생성합니다.

핵심 직관: 신경망 전체는 결국 하나의 함수 idx → logits 입니다. 입력 idx는 모양이 [batch, time]인 토큰 id 격자이고, 출력 logits는 모든 위치에서 모든 어휘 단어에 대한 점수로 모양은 [batch, time, vocab] 입니다. 나머지는 전부 이 함수를 둘러싼 배관 작업일 뿐입니다.

참고: README는 이 저장소를 후속작 nanochat에 자리를 내준 오래된/지원 종료 저장소로 표시합니다. 그래도 GPT의 동작 원리를 배우기에는 nanoGPT가 가장 깔끔하며, 이 페이지의 목적이 바로 그것입니다.

구성요소 1 — CausalSelfAttention

토큰들이 앞선 토큰들을 "바라보는" 곳입니다. 인과적(Causal)이란 토큰이 자기 자신과 과거만 볼 수 있고 미래는 절대 볼 수 없다는 뜻입니다(그렇지 않으면 다음 토큰 예측에서 부정행위가 됩니다).

model.py
  • 한 번의 행렬곱으로 Q, K, V 생성. c_attn이 각 토큰 벡터(C)를 3C로 투영한 뒤 query/key/value로 분리합니다.
  • 헤드로 재배열. C가 hs = C/nh 크기의 nh개 헤드로 쪼개져 모양이 [B, nh, T, hs]가 되고, 각 헤드는 독립적으로 어텐션합니다.
  • 어텐션 점수. q @ kᵀ가 [B, nh, T, T] 행렬을 만듭니다 — 각 토큰이 다른 토큰에 얼마나 주목하는지. 1/√hs로 스케일링하고, 인과 마스킹 후, 소프트맥스를 적용합니다.
  • value의 가중합. att @ v → [B, nh, T, hs], 이후 헤드들을 다시 [B, T, C]로 이어 붙이고 c_proj로 투영합니다.
  • Flash attention(scaled_dot_product_attention, PyTorch ≥ 2.0)은 점수+마스크+소프트맥스+행렬곱을 하나의 빠른 커널로 융합합니다 — 수학은 동일하되 메모리에 명시적 [B,nh,T,T] 텐서를 만들지 않습니다.

8 · 추론 / 샘플링

생성은 순전파를 감싼 루프입니다. 매 스텝: 다음 토큰 logits를 예측하고, 토큰 하나를 샘플링하고, 이어 붙이고, 반복합니다. 이것이 자기회귀(autoregressive) 디코딩입니다.

model.py
  1. 컨텍스트 자르기 — 마지막 block_size 토큰까지만(모델은 그보다 더 과거를 못 봄).
  2. 순전파 → 마지막 위치의 logits만 취함: logits[:, -1, :], 모양 [B, V].
  3. 온도(temperature)가 logits를 나눔: <1이면 날카롭게(안전), >1이면 평평하게(과감).
  4. top-k — 샘플링 전에 가장 가능성 높은 k개를 제외한 나머지를 0으로.
  5. 소프트맥스 → 다항 샘플링 → 새 토큰 id 하나를 시퀀스에 이어 붙임.
generate()의 @torch.no_grad()는 그래디언트 추적을 끕니다 — 추론은 학습하지 않으므로, 역전파 부기(bookkeeping)를 건너뛰어 메모리와 시간을 아낍니다.

sample.py가 이를 감쌉니다: 체크포인트를 불러오고, meta.pkl(문자 단위)에서 인코더/디코더를 재구성하거나 tiktoken의 GPT-2 BPE로 폴백하고, start 프롬프트를 인코딩하고, generate()를 호출한 뒤, id를 다시 텍스트로 디코딩합니다.
